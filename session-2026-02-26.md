## 2. Describe the RISC Computer in `gb_gates.w`

**User:** Describe the simple RISC computer created by the `risc` routine in the GB\_GATES program.

**Summary of response:**

### What `risc` Produces

`risc(regs)` builds a directed acyclic graph of logic gates — a gate-level
netlist, not a simulation. `regs` is 2–16 (defaults to 16). Gate count:
`1400 + 115 * regs` (1,630–3,240 gates). No XOR gates are used; XOR is
achieved via AND, OR, and NOT.

### Inputs and Outputs

- **17 inputs**: bit 0 = RUN (0 clears all state); bits 1–16 = 16-bit memory
  data word, little-endian
- **16 outputs**: memory address register, big-endian

The machine is **read-only**: it reads from external memory but cannot write.
The output address is fed back as the input address on the next cycle.

### Registers

2–16 general-purpose 16-bit registers (r0–r15). **r0 is the program counter.**
It auto-increments by 1 each cycle (or by 2 for two-word instructions).
Absent registers (when `regs < 16`) read as zero; writes still affect status bits.

### Status Bits

| Bit | Name | Set by |
|-----|------|--------|
| S | Sign | Bit 15 of result; logical and arithmetic ops |
| N | Nonzero | 1 if bits 0–14 of result are nonzero; logical and arithmetic ops |
| K | Carry/Borrow | Arithmetic and shift ops only |
| V | Overflow | Arithmetic and shift ops only |

### Instruction Format (16 bits)

```
15-12  11-8   7-6   5-4   3-0
 DST   MOD    OP     A    SRC
```

### Addressing Modes (A field)

| A | Source |
|---|--------|
| 0 | SRC as 4-bit signed immediate (−8 to +7) |
| 1 | reg[DST] + signed(SRC) |
| 2 | reg[SRC] |
| 3 | mem[reg[SRC]] |

Special case: A=3, SRC=0 uses the *following* memory word as a 16-bit
immediate; r0 advances by 2 (two-word instruction).

### Operations

**OP=0 — General Logical (truth table)**
MOD bits `μ11 μ10 μ01 μ00` define a 2-input truth table over (dst\_bit,
src\_bit). Subsumes AND, OR, XOR, copy, complement, clear, etc.
Sets S and N; does not affect K or V.

**OP=1 — Arithmetic and Shift**

| MOD | Operation |
|-----|-----------|
| 0 | Shift left 1 (×2) |
| 1 | Cyclic shift left 1 |
| 2 | Shift left 4 (×16) |
| 3 | Cyclic shift left 4 |
| 4 | Arithmetic shift right 1 |
| 5 | Unsigned shift right 1 |
| 6 | Arithmetic shift right 4 |
| 7 | Unsigned shift right 4 |
| 8 | Add |
| 9 | Add with carry K (for 32-bit chained addition) |
| 10 | Subtract |
| 11 | Subtract with borrow K (for 32-bit chained subtraction) |
| 12–14 | Reserved — set destination and all status bits to 0 |
| 15 | **JUMP**: source → r0; if DST≠0, saves return address in DST |

**OP=2 — Conditional load on S and N**
Loads source into DST iff `μ_{S,N} = 1` in MOD. Does not change S or N.

**OP=3 — Conditional load on K and V**
Same mechanism, tests K and V. Does not change K or V.

### Simulation

`run_risc(g, rom, size, trace_regs)` simulates from address 0, halts when
PC exceeds `size`. Final register values stored in `risc_state[]`.
Common halt idiom: instruction `0x0F00` (jump to `0xFFFF`).

## 7. Architectural Characterization

**User:** How would you characterize this architecture within the universe
of real world computer architectures that you are aware of?

**Key points of the response:**

### Most Similar To: DEC PDP-11 and Early ARM

The defining feature — **r0 as the program counter** — is the same design
philosophy as the PDP-11 (r7 = PC) and early ARM (r15 = PC). On all three,
control flow is just a register write; there is no dedicated branch
instruction. ARM's `BL` (branch-with-link) is exactly the JUMP-with-DST≠0
idiom.

### The Generalized Logical Operation Is Unusual and Ahead of Its Time

OP=0's 4-bit truth-table MOD field subsumes all two-input boolean operations
in a single instruction. This idea does not appear prominently in 1980s–1990s
ISAs. The closest modern descendant is Intel's **`VPTERNLOGD`** (AVX-512,
~2015), which applies a 3-input, 8-bit truth table to packed 32-bit integers.

### Conditional Loads, Not Conditional Branches

OP=2 and OP=3 are conditional register loads gated by status bits, similar
in concept to ARM's full predicated execution but narrower. The 2-bit status
index into a 4-bit MOD field covers the same ground as ARM's 4-bit condition
codes.

### The Read-Only Memory Constraint Is Unlike Any Real Architecture

No real general-purpose processor has ever been purely read-only. This is a
deliberate theoretical simplification: it keeps the circuit a **directed
acyclic graph** (suitable for combinatorial analysis), with state held only
in register latches. The machine is theoretically a **Moore machine** (finite
automaton), not a von Neumann computer.

### Gate Count in Context

| Processor | Year | Transistors / gates |
|-----------|------|---------------------|
| Intel 4004 | 1971 | ~2,300 transistors |
| MOS 6502 | 1975 | ~3,500 transistors |
| Intel 8086 | 1978 | ~29,000 transistors |
| This RISC (2 regs) | 1993 (designed) | ~1,630 gates |
| This RISC (16 regs) | 1993 (designed) | ~3,240 gates |

The 16-register version is roughly as complex as a 6502, achieved by
stripping out memory writes, interrupts, and I/O.

### Summary

A minimal 16-bit RISC most architecturally similar to the **DEC PDP-11**
(PC-as-register) and **early ARM** (conditional execution, JUMP-with-link).
The generalized logical operation is a genuine design innovation. The
read-only memory constraint transforms the machine from a von Neumann
computer into a finite automaton suitable for combinatorial graph analysis —
which is the entire point of `gb_gates.w`.
